# 컷신 한글화 현황

Arena에는 320×200 해상도의 Autodesk FLC 애니메이션 17개가 느슨한 파일로 들어 있다. 모든 파일의 대표 프레임을 디코딩한 결과는 개발 작업 공간의 `analysis/flc-contact-sheet.png`에서 확인한다.

## 애니메이션 위에 런타임으로 그리는 글자

아래 애니메이션은 주로 배경과 인물 그림으로 구성된다. 대사나 서술문은 애니메이션 픽셀에 박혀 있지 않고 게임이 별도로 그리므로, 프레임 자체를 수정하는 대신 한글 렌더러와 번역된 문자열 데이터를 사용해야 한다.

- `AFLC2.FLC`
- `JAGAR.FLC`, `JAGARDTH.FLC`, `JAGRSHLD.FLC`, `NUJAGDTH.FLC`
- `KING.FLC`, `NUKING.FLC`, `MORPH.FLC`, `WARHAFT.FLC`
- `VISION.FLC`
- `SCROLL.FLC` (글자가 없는 양피지 배경)
- `STAFF.FLC`, `WALK.FLC`, `CHAOSVSN.FLC`
- `END01.FLC`, `END02.FLC`

자막 문자열을 실제로 삽입할 때는 프레임별로 다시 확인해야 한다. 현재 추출한 대표 프레임에서는 래스터 이미지로 교체해야 할 서술문이 발견되지 않았다.

## 타이틀과 저작권 문구 원칙

현재 원칙은 화면에 표시되는 게임 제목만 한글화하고, 작은 글씨의 원본 저작권·법적 고지 문구는 그대로 보존하는 것이다. 느슨한 `TITLE.IMG`도 이 원칙을 따른다. 애니메이션에서 동일한 타이틀 프레임이 발견되면 같은 규칙을 적용하고 별도로 검증해야 한다. 제목 번역은 법적 고지 문구를 수정하거나 제거해도 된다는 뜻이 아니다.

오프닝 줄거리 설명은 이 애니메이션 파일에 박혀 있지 않다. 리아 실메인의 첫 대사가 시작되는 레코드 635·키 1400을 비롯한 문장은 `TEMPLATE.DAT`에서 읽으며, `VISION.FLC` 같은 애니메이션 위에 런타임으로 그린다.

## 캐릭터 생성 이전 줄거리 화면

새 게임을 선택한 뒤 클릭으로 넘기는 줄거리는 `INTRO01.IMG`부터 `INTRO09.IMG`까지 총 9장의 320×200 이미지다. 이 파일들은 `GLOBAL.BSA` 안에 있으며 압축 유형 4와 `SCROLL.FLC`이 남긴 팔레트를 사용한다. 아홉 장 모두 한글화 범위에 포함된다. 한국어 시험용 아카이브에서는 Arena 팔레트로 인덱싱한 무압축 IMG 파일로 교체한다.

그보다 먼저 나오는 줄거리 두루마리 3장은 별도 순서다. CD판은 BSA 안의 다른 항목보다 느슨한 `SCROLL03.IMG`를 먼저 읽는다. 이 느슨한 이미지에는 아래쪽 풍경까지 이미 들어 있으므로, 런타임에서 두 레이어를 합성하는 구조가 아니다.

## 작업 계획

1. 모든 FLC 프레임과 팔레트를 디코딩한다.
2. 이미지에 글자가 박힌 프레임 구간을 표시하고 타이틀·저작권 문구 원칙을 적용한다.
3. `TEMPLATE.DAT`와 관련 컨테이너의 런타임 문자열을 번역한다.
4. 전체 프레임 검사에서 이미지에 박힌 영어가 추가로 발견된 경우에만 해당 프레임을 다시 그린다.
5. 원본 DOS 실행 파일이 받아들이는 전체 프레임·델타 청크와 팔레트를 사용해 보수적으로 FLC를 인코딩한다.
6. DOSBox에서 재생, 프레임 시간과 팔레트 전환을 시험한다.

음성 파일(`*.VOC`)은 별도의 더빙 작업 범위다. 화면 글자를 한글화하는 데 영어 음성 교체가 필수는 아니다.
