# 변경 기록

- 원본 텍스트 카탈로그 1,934개를 전수 분류하고 번역 대상 1,869개 전체에 1차 번역을 적용했다. `QUESTION.TXT` 1개는 전용 한글 파일로, 비언어 제어값 64개는 원형 그대로 처리한다.
- 55개 BSA 내부 INF와 11개 느슨한 텍스트 파일을 재구성하고, 원본 바이트 위치가 고정된 해시 레코드 103개를 크기 안에 재조판했다. 자리표시자·AKC 문자·BSA 항목·CITYINTR 64개 구역 식별자를 빌드에서 검증한다.
- 잔존 로마자를 자동 감사해 영어 직업명, 도시 내부 식별자, 수수께끼 입력 정답, 엔진 표식과 지도·퍼즐 코드만 남기고 일반 표시 문자열의 미분류 영문을 제거했다.
- v0.5.0 설치기 델타 대상에 아티팩트, 도시 진입, 던전, 장비, 마법사 길드, 상점, 주문 제작과 주점 데이터 9종을 추가했다.
- 저장소 최상단에 `Installer.bat`과 완전한 `patcher/` 배포 자원을 배치해 GitHub의 `Download ZIP`으로 받은 저장소도 즉시 설치할 수 있게 했다. 같은 설치기로 Steam 설치 경로 자동 탐색, 원본 해시 검사, 진행률 표시, 한글 런타임 설치, 상태 검사, 원본 실행 설정 복구와 GitHub Release 자동 업데이트를 수행한다. 설치는 원본 `ARENA`를 수정하지 않고 `ARENA_KR`을 만들며, 실패 시 트랜잭션 백업으로 자동 롤백하고 복구 시 세이브 보호를 위해 `ARENA_KR`을 보존한다.
- 원본 전체 파일을 배포하지 않도록 `arena-korean-delta-v1` 델타 생성·적용 형식과 릴리스 빌더를 추가했다. v0.3.0 배포 ZIP은 9개 파생 파일의 델타, 5개 payload, 3종 한글 폰트의 저작권·OFL·출처 문서 및 ZIP SHA-256 파일로 구성한다.
- 공개 저장소에 포함되는 Galmuri9 파생 컷신 상주 글리프 캐시에 원저작권 문구, OFL 1.1 전문, 원본 버전·해시와 변환 내역을 함께 추가했다.
- 고정 오프셋 키 1400·1402 격리판의 실기 시험에서 첫 조각 획득 전 사망 자막이 리아 실메인의 한국어 키 1402로 정상 출력됨을 확인했다. 진행값 우회 없이 해시 레코드 바이트 위치 보존만으로 해결됐다.
- 리아 영상 위에 `.Perhaps I will even...`이 나온 원인은 키 1403 분기가 아니라 `TEMPLATE.DAT`의 고정 본문 오프셋 파괴였다. 짧아진 한글 키 1400 때문에 원본 키 1402 본문 주소 `0x499A0`이 한글 파일에서는 키 1403의 `. Perhaps I...` 위치가 됐음을 바이트로 확인했다.
- 모든 해시 레코드 번역을 원본 바이트 크기에 맞추도록 빌더를 수정했다. 표시 종료표식 `&` 뒤에 공백을 채우고 전체 파일 크기와 811개 헤더의 키·오프셋이 원본과 정확히 같은지 검증한다. 고정 오프셋 키 1400·1402 시험 템플릿은 395,981바이트, SHA-256 `82c3348df9ca9ec78189c5ce50216a204b5c7e8dbac4eb973cf9266d3f35f2c0`다.
- 원문과 같은 33행 격리판에서도 같은 잘못된 본문이 나와 34번째 행·12번째 자막 구간과 단순 행 버퍼 초과를 배제했다. 새 게임 뒤 `P=0` 안전 가드도 결과를 바꾸지 못해 철회했으며, 관련 훅을 제거한 `ACDKR.EXE`는 가드 전 검증본 SHA-256 `d50e8e66afb5bd7ffb35d1d206ac2e4a79e04af022fbd72dd86b3499dc781768`과 다시 일치한다.
- 원본 영어 템플릿 A/B 판에서 두 장면이 정상인 이유는 ASCII 렌더러 차이가 아니라 원본 레코드 바이트 위치가 유지되기 때문으로 확정했다. 키 1400 번역은 원문과 같은 33행·11구간을 계속 유지하며, 모든 컷신 키의 원문·번역 행 수 일치도 빌드에서 강제한다.
- 저장 상태 초기화 뒤에도 리아 영상 위의 키 1403 후반 본문이 재현되어 상태 오염을 배제했다. `ACDKR.EXE`와 TSR은 유지하고 `TEMPL_KR.DAT`만 원본 영어 데이터로 바꾼 A/B 판을 기존 `ARENA_KR`에 배치했다.
- 별도 클린 실행판은 새 게임 초기화용 `.65` 파일까지 빠져 검은 화면에서 멈춘 것을 확인하고 폐기했다. 기존 `ARENA_KR`의 세이브·활성 슬롯·임시 파일 31개를 백업 후 제거하고, 저장 목록을 `NAMES.CLR`의 전 슬롯 `EMPTY` 상태로 초기화했다.
- 키 1400·1402만 담은 203개 한글 진단판에서도 첫 조각 전 리아 사망 영상 위에 `years as Emperor`가 포함된 키 1403 후반 본문이 읽힘을 확인했다. 기존 세이브를 보존하면서 원인을 가리기 위해 가변 저장·임시 파일을 제외한 `ARENA_KR_CLEAN_TEST`와 전용 실행 배치를 만들었다.
- 원본 영어판 실기 대조에서 새 게임·첫 지팡이 조각 획득 전 사망은 `With you died our...`로 시작하는 리아 키 1402가 맞음을 확인했다. 이전 한글판의 키 1403 호출은 원작 동작이 아닌 회귀로 확정하고 한글판 동일 조건 재시험 항목으로 기록했다.
- 첫 지팡이 조각 획득 전에는 리아 키 1402가 나와야 한다는 지적에 따라 원본을 재검증했다. 새 게임 `P` 초기값은 0이고 원본·패치본의 사망 분기도 동일하므로 키 1403 시험판을 철회하고, 키 1400·1402만 담은 203개 캐시 진단판을 배치했다.
- 192개 캐시 기준선에서 첫 리아 장면과 영어 사망 자막이 모두 정상 출력되는 것을 확인했다. 첫 조각 전 리아 영상에 제이거 탄 본문이 나온 것을 당시 키 1403 호출로 오분류해 문서를 임시 수정하고 키 1400·1403 단일 시험판을 배치했으나, 이후 고정 본문 오프셋 문제로 바로잡았다.
- 리아 실메인 첫 장면이 앞의 6행을 건너뛰고 진행 전 사망 장면이 깨지는 회귀를 338개 상주 캐시판에서 확인했다. 키 1400 본문과 영상은 정상판과 같고 글리프 바이트도 일치하므로, 실행본을 키 1400 전용 192개 캐시 기준선으로 되돌려 분리 시험한다.
- 바닥·상자·시체의 공용 아이템 목록을 HANGUL9 기준 11픽셀 간격의 두 항목과 별도 `...More` 행으로 재배치하고, 수량 입력 커서도 같은 행 격자에 맞췄다.
- 빈 시체·상자 문구를 `쓸 만한 것이 없다`로 다듬었다.
- 리아 실메인·제이거 탄 관련 21개 장면의 발동 조건·자원·키·행 수·상태와 한국어 대사 전체를 번역 JSON에서 자동 생성하는 문서와 검증 도구를 추가했다.
- 자동지도의 `N/E/S/W`, `Exit` 원본·클린본·개별 한글 PNG·실제 크기 아틀라스·배치 좌표 JSON 작업 묶음과 팔레트·허용 영역 검증 가져오기 도구를 추가하고, 사용자 완성본을 `AUTOMAP.IMG`로 반영했다.
- 훔치기 대상 선택 안내, 현재 상태창의 위치·시각·날짜·소지 중량, 탐리엘 달력의 달·요일·시간대, 자동지도 첫 던전 제목을 한글화하고 자동지도의 고정 이미지·동적 출력 구성을 문서화했다.
- 장비 화면 완성본의 `레벨:`·`장비`·`종료`·`마법서`·`버리기` 최종 좌표를 반영하고, 팔레트 외 색과 비글자 픽셀 변경을 거부하는 전용 가져오기 도구로 `EQUIP.IMG`를 적용했다.
- 이미지 고정 글자 작업 묶음은 개별 투명 PNG와 실제 크기 아틀라스·좌표 JSON을 항상 함께 만들도록 규칙과 생성 도구를 보완했다.
- 첫 본편 리아 실메인 꿈의 키 1500 혼돈의 지팡이 설명과 키 1294 팽 레어 안내를 한글화하고, 원문과 같은 15행·16행 화면 구성을 유지했다.
- `CHAOSVSN.FLC`의 자막 밴드를 Y=163~199로 확장하면서 위쪽 그림·31프레임·팔레트·재생 속도를 보존하고, 새 컷신 음절을 TSR 상주 HANGUL9 캐시에 포함했다.
- 열쇠 재질 12종의 영문 관사를 제거해 `a Ruby 열쇠` 같은 혼합 표기를 `루비 열쇠`로 수정했다.
- 몬스터 23종, 무기·방어구·장비 부위·재질·물약·마법 접미사·내구 상태 등 런타임 이름 135개를 한글화했다. 같은 표 뒤의 직업명 19개는 합의한 영어 표기를 유지한다.
- 장비 화면 `EQUIP.IMG`의 `Level`, `EQUIPMENT`, `EXIT`, `SPELLBOOK`, `DROP`을 `레벨`, `장비`, `종료`, `마법서`, `버리기`로 교체하고, 동적 `상태·피해·방어도·무게·충전·사용 횟수` 문구를 한글화했다.
- 첫 던전 `START.INF` 환경 설명 6개와 문·자물쇠·열쇠·상자·금화·인벤토리·장비·휴식·야영·저장·이동·기초 주문·장비 상세 공통 메시지 125개를 한글화했다. 실행 파일 문구는 원문 바이트 예산과 자리표시자를 빌드할 때 검증한다.
- 자동지도와 일지 이미지의 `N/E/S/W`, `Exit`, `Journal`, `print`를 `북/동/남/서`, `종료`, `일지`, `인쇄`로 교체했다.
- 리아 실메인 자막의 검은 밴드를 33행에서 37행으로 확장하고, 세 줄을 Y=166·177·188의 11픽셀 간격으로 배치했다.
- 리아 실메인 첫 꿈 장면을 당시 34행·12구간으로 조판했으나, 이후 원문은 33행·11구간임을 확인해 바로잡았다.
- 컷신 템플릿의 ASCII 허용 필터가 AKC 바이트를 삭제해 이름만 보이고 한글이 점처럼 남던 문제를 수정했다. 마지막 소문자 범위 분기를 signed 비교로 바꿔 AKC 선행·후행 바이트를 보존한다.
- 지역 선택 뒤 혈통 안내문의 임시 버퍼 조립·40바이트 자동 줄바꿈 경로를 우회하고, 8개의 완성된 한글 설명을 앞뒤 안내문과 같은 공용 출력기로 직접 표시하도록 수정했다. 상자 높이 보정은 혈통 설명에만 적용한다.
- 혈통 상자 높이용 후행 빈 줄을 모두 제거하고, 혈통 출력 중에만 측정 높이를 `n×11`로 계산하도록 전용 플래그와 도우미를 추가했다.

## 미배포

- 리아 실메인 진행 전 사망 키 1402 단일 시험은 338개 회귀 캐시를 그대로 바탕으로 만들어 원인 분리가 되지 않았고, 첫 장면 건너뜀과 사망 장면 깨짐이 재현되어 철회했다.
- 리아 실메인의 진행 꿈 키 1295~1302와 제이거 탄의 방해 키 1392~1399·P=1~8 사망 키 1403은 혼선 보고 뒤 격리했다. 현재 실행본은 키 1400과 P=0 사망 키 1402만 포함한 203개 캐시 진단판이며, 호출 매트릭스 순서대로 개별 재시험한다.
- 메인 메뉴 이미지를 조선100년체 기반 최종 작업물로 교체하고 세 클릭 범위를 실제 글자 Y좌표에 맞춤
- 메뉴 픽셀을 `SCROLL.FLC` 팔레트로 양자화하고 다른 내장 팔레트를 붙여 인게임 색이 뒤틀리던 문제 수정
- 8비트 `CHARSHT.COL`을 6비트 VGA 팔레트로 잘못 확대해 스탯창 추출 PNG가 밝은 청록색이 되던 문제 수정
- 능력치 배분 화면의 최종 `CHARSTAT.IMG`, `BONUS.IMG` 작업물을 적용하고 `BONUS.IMG`를 `(62,119)`로 이동
- 기본 능력치 수치·선택 화살표·클릭 영역·부분 갱신 영역을 11픽셀 행 간격으로 동기화
- 능력치 화면 용어를 `인내`, `활력`, `잔여 스탯`, `성장 체력`, `카리스마` 등으로 확정
- 능력치명과 종족명 실행 파일 문자열 표, 저장·다시 굴림, 잔여 보너스 경고와 외형 선택 안내를 번역
- 지역 확정 뒤 공통 안내문 4개, 종족별 설명 8개, 직업별 선호 능력치 문구 18개를 번역해 능력치 배분 화면 직전까지 한글화
- 한글 `OUTPROV.CIF`를 `GLOBAL_K.BSA`에 다시 적용하고 현재 지역 표시용 금색 지역명 9개 복구
- `OUTPROV.CIF`의 용도를 캐릭터 생성 지도 호버가 아닌 생성 이후 월드맵의 현재 지역 표시로 정정
- 원본 실행과 한글 실행이 느슨한 이미지 파일을 공유하지 않도록 한글 전용 로컬 런타임 `ARENA_KR` 구조를 도입
- `Arena Korean Test (Windowed)`가 `ARENA_KR`만 마운트하도록 변경
- 원본 `ARENA`의 `TITLE.IMG`, `SCROLL03.IMG`, `TAMRIEL.MNU`를 정품 상태로 복원하도록 개발 설치 구조 수정

이 프로젝트는 공개 릴리스 전 개발 단계다. 날짜는 한국 표준시를 기준으로 기록한다.

### 변경

- 게임 제목은 한국어로 표시하고 원본 저작권 문구는 유지
- `SCROLL03.IMG`는 실제 CD판 느슨한 파일을 기준으로 처리
- 활성 영문 글꼴 높이에 맞추던 한글 축소와 최대 8픽셀 제한을 폐기하고 Galmuri9 9×9 원본 출력 적용
- 한글이 있는 줄에 최소 11픽셀 행 이동값을 반환하는 `INT 60h`, `DX=2` 처리 추가
- 직업 질문 줄바꿈을 `FONT_B` 실제 폭표와 원본 299픽셀·126픽셀 배치 한계로 검증하도록 변경
- 질문 이어줄의 강제 공백 들여쓰기 제거
- 한글 혼합 줄에서 영문 글리프를 한글 9픽셀 셀의 아래쪽 기준선에 맞추도록 보정
- 판정 태그만 남는 ASCII 전용 이어줄을 금지해 질문 전체의 11픽셀 행간 유지
- 원본과 다른 CRLF 및 후행 공백 누락 때문에 문항 구분과 정렬이 바뀌던 문제를 수정하고 `QUESTION.TXT`와 동일한 `공백 + LF` 줄 끝 적용
- 원문의 질문·선택지 구분자 `:`을 복원하고 엔진과 동일한 문항 경계 검증 추가
- 원본 그리기 함수처럼 NUL 소비 뒤 `SI`를 다음 문자열로 전진시켜 질문 스크롤 초기 화면이 비던 문제 수정
- 질문 두루마리 전용 8픽셀 격자를 HANGUL9 기준 11픽셀 격자로 전환하고 스크롤 버퍼·임계값 동기화
- 질문 본문 오른쪽에 10픽셀 여백을 확보하도록 줄 폭 한계를 290픽셀로 조정
- 한글을 영문 폰트 높이에 맞춰 축소하지 않고 9·12·16픽셀 원본 뱅크와 화면별 UI 조정을 사용하는 정책 확정
- 폰트 후보의 OFL 1.1 조건, 예약 글꼴명과 배포 고지 구조 문서화
- 영문 글꼴 높이에 따라 HANGUL9·HANGUL12·HANGUL16 원본 뱅크를 선택하는 TSR 구현
- 물마루 Mono 12와 Neo둥근모 1.601에서 완성형 한글 11,172자를 Arena용 원시 뱅크로 생성하는 도구 추가
- 357,504바이트 폰트의 마지막 EMS 페이지를 16KiB 완전 페이지로 잘못 검사해 TSR 설치가 실패하던 문제 수정
- 물마루 Mono의 원본 12픽셀 전각 셀에 간격을 중복 추가하던 HANGUL12 전진폭을 13에서 12로 수정
- 물마루 PFP의 축약 모음·종성 컴포넌트를 위쪽에 합성해 아래 획이 사라지던 문제를 기준선 아래 정렬로 수정
- 질문 종료 후 추천 직업 결과 문구 번역, 직업명 18개는 영어 원문 유지
- 추천 결과의 `Yes / No`가 공용 대화상자의 실행 파일 문자열임을 확인하고 `예 / 아니요` 포인터 패치 구현
- 잘못 준비했던 `YESNO.IMG` 이미지 작업 경로 제거
- 잘못 연결했던 `ACCPREJT.IMG` 교체를 원본으로 복구
- 직접 직업 선택 제목, 이름 입력 안내, 성별 선택 제목을 지도 진입 전 범위까지 한국어로 교체
- 이미지 작업물은 실제 적용 해상도만 보관하고 혼동을 일으키는 확대 미리보기 복제본을 생성하지 않도록 변경
- 출신 지역 지도 9개 기본명·강조명을 HANGUL9로 제작하고 사용자 확정 좌표 적용
- `TAMRIEL.MNU`의 고정 오프셋 클릭 마스크를 보존하는 type-4 지도 재압축 도구와 한글 `OUTPROV.CIF` 빌더 추가
- 지도 압축 뒤 0 패딩을 디코더가 추가 명령으로 읽어 검은 화면에서 멈추던 문제를 수정하고, 유효한 type-4 토큰만으로 원본 길이를 정확히 채우도록 변경

### 알려진 문제

- 능력치 화면의 새 10픽셀 행 좌표와 클릭 판정은 실기 검증 대기
- 직업 추천 결과와 직접 선택 목록은 아직 영어
- 첫 던전의 새 몬스터·아이템 이름과 장비 화면은 런타임 시험 대기이며, 도시 NPC·상점·퀘스트 본문은 미번역
- 한글 이름 입력은 지원하지 않음
- 배포용 단일 CMD 패처는 설계 단계
