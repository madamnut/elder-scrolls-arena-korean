# 한글 폰트 정책과 라이선스

이 문서는 Arena 한국어 패치에서 사용할 한글 비트맵 폰트의 크기 정책, 영문 폰트 대응 관계, 후보 선정 상태와 배포 의무를 기록한다. 폰트 정책을 바꿀 때는 이 문서와 변경 기록을 함께 갱신한다.

## 1차 결정

2026년 7월 11일 다음 원칙을 1차 정책으로 확정했다.

1. 한글 글리프를 활성 영문 폰트 높이에 맞춰 임의로 축소하지 않는다.
2. 한글의 최소 표시 높이는 9픽셀로 둔다.
3. 필요한 크기는 처음부터 해당 크기로 제작한 비트맵을 사용한다.
4. 한글이 기존 영역에 들어가지 않으면 폰트를 줄이지 않고 UI 박스, 출력 좌표, 행간과 줄바꿈 폭을 수정한다.
5. 그래도 공간이 부족하면 번역을 간결하게 다듬거나 내용을 여러 화면으로 나눈다.
6. 영문과 한글은 같은 화면에서 서로 다른 높이를 사용할 수 있다.
7. 원본 영문 글꼴과 출력은 훼손하지 않는다.

2026년 7월 11일 1차 구현에서 활성 영문 글꼴 높이에 맞추던 최근접 축소 코드를 제거했다. 2026년 7월 12일에는 영문 글꼴 높이에 따라 9·12·16픽셀 원본 뱅크를 선택하는 기능을 구현했다. 어떤 뱅크도 실행 중 축소하지 않는다.

## 글자 및 행간 규칙

각 한글 뱅크는 `글리프 크기`, `가로 전진폭`, `행 이동값`을 분리해서 관리한다. 글리프는 원본 픽셀을 축소하지 않고, 가로 전진폭은 다음 글자의 시작점, 행 이동값은 다음 줄의 시작점을 뜻한다.

| 뱅크 | 글리프 | 가로 전진폭 | 글자 사이 빈 칸 | 행 이동값 | 줄 사이 빈 행 | 상태 |
|---|---:|---:|---:|---:|---:|---|
| `HANGUL9.FNT` | 9×9 | 10 | 1 | 11 | 2 | 구현·실기 통과 |
| `HANGUL12.FNT` | 12×12 셀 | 12 | 셀 내부 포함 | 14 | 2 | 구현·실기 검증 대기 |
| `HANGUL16.FNT` | 16×16 셀 | 17 | 1 | 18 | 2 | 구현·실기 검증 대기 |

공백과 ASCII 문장부호는 현재 활성화된 원본 영문 폰트의 폭표를 그대로 사용한다. 혼합 문자열의 폭 계산과 실제 그리기 전진폭은 반드시 같아야 하며, 줄바꿈은 글자 수가 아니라 이 픽셀 폭으로 결정한다. 가운데 정렬되는 화면에서는 자동 이어줄 앞에 공백을 넣지 않는다.

가로와 세로의 기준은 분리한다.

1. 글자 사이 간격은 각 문자를 그린 폰트의 전진폭을 따른다. 영문·숫자·ASCII 문장부호와 공백은 활성 영문 폰트의 가변폭을 사용하고, 한글 음절은 선택된 한글 뱅크의 고정 전진폭을 사용한다.
2. 줄간격은 한글화된 텍스트 블록에 배정된 한글 뱅크의 행 이동값을 따른다. 같은 블록 안에 영문과 한글이 섞이거나 ASCII만 있는 줄이 끼어도 행간을 줄마다 바꾸지 않는다.
3. 원본 영문만 표시하는 화면은 원본 `font[0] + 1` 행간을 그대로 유지한다.
4. 혼합 줄의 세로 정렬은 별도 기준선 보정값으로 다룬다. 영문 글자의 Y 위치를 조정하더라도 블록의 행 이동값은 바꾸지 않는다.

따라서 질문 화면은 `FONT_B`의 문자별 가변폭과 `HANGUL9`의 10픽셀 전진폭을 함께 사용하지만, 모든 질문 줄의 행 이동값은 `HANGUL9` 기준 11픽셀로 통일한다.

질문 두루마리는 일반 문단 행간 함수와 별개의 8픽셀 스크롤 격자를 사용하므로 실행 파일의 전용 상수도 11픽셀로 패치한다. 글리프 9행 뒤에 빈 행 2개가 생기며, 스크롤도 한 논리 줄당 11단계로 이동한다.

질문 텍스트는 X=20에서 시작하고 최대 폭을 290픽셀로 제한해 오른쪽 X=310 이후에 10픽셀 여백을 둔다. 글자를 축소하지 않고 줄바꿈 줄 수와 스크롤 범위가 늘어나는 방식으로 처리한다.

혼합 줄의 기준선은 한글 셀의 아래쪽을 기준으로 맞춘다. HANGUL9가 포함된 줄에서 높이가 9픽셀보다 작은 영문 글꼴은 `9 - 영문 글꼴 높이`만큼 아래로 이동한다. 따라서 높이 6픽셀인 `FONT_B`는 질문 화면에서 3픽셀 내려가며, 영문의 가로 전진폭과 줄 전체의 11픽셀 행 이동값은 변하지 않는다.

## 한글 뱅크 후보

| 한글 뱅크 | 1차 후보 | 현대 한글 11,172자 | 라이선스 | 상태 |
|---|---|---:|---|---|
| `HANGUL9.FNT` | Galmuri9 | 전부 확인 | OFL 1.1 | 구현 완료, 질문 화면 실기 통과 |
| `HANGUL12.FNT` | 물마루 Mono 12 | 전부 확인 | OFL 1.1 | 구현 완료, 화면 비교 필요 |
| `HANGUL16.FNT` | Neo둥근모 1.601 | 전부 확인 | OFL 1.1 | 구현 완료, 화면 비교 필요 |
| `HANGUL16.FNT` | GNU Unifont 16 | 전부 확인 | OFL 1.1 또는 GPL 2+와 폰트 예외 | 대체 후보 |

Ark Pixel Font 10px 한국어 BDF는 OFL 1.1이지만 완성형 `가–힣` 11,172자가 직접 들어 있지 않아 현재 AKC 글리프 뱅크 입력으로 사용하지 않는다.

물마루 PFP의 초성 컴포넌트는 12행 전체 셀을 사용하지만 일부 가로 모음과 종성 컴포넌트는 위쪽 투명 행을 생략한 5~7행 데이터다. 합성기는 이 축약 컴포넌트를 12행 셀의 기준선 아래쪽에 맞춘 뒤 겹친다.

후보 원본은 개발 작업 공간의 `reference/font-candidates/`에 보관한다. 이 디렉터리는 공개 저장소와 배포물에 통째로 포함하지 않는다.

## 실제 파일 구조

한글 글리프 레코드는 16개의 16비트 행, 총 32바이트다. 따라서 9×9뿐 아니라 최대 16×16 글리프도 레코드 크기를 바꾸지 않고 저장할 수 있다.

```text
글리프 1개 = 16행 × 16비트 = 32바이트
한글 뱅크 1개 = 11,172자 × 32바이트 = 357,504바이트
EMS 페이지 1개 = 16KiB = 글리프 512개
```

세 뱅크는 각각 22페이지이며 TSR 설치 때 총 66 EMS 페이지(1,081,344바이트)를 할당해 동시에 적재한다. 글리프를 그릴 때는 현재 영문 글꼴 높이가 9 이하이면 9픽셀, 10~12이면 12픽셀, 13 이상이면 16픽셀 뱅크를 선택한다. 품질을 위해 런타임 축소를 다시 도입하지 않는다.

## 라이선스 결론

### 비용과 사용 범위

선정 후보는 모두 개인·상업·비상업 용도로 사용할 수 있고 게임이나 패처에 묶어 배포할 수 있다. 별도 사용료나 사전 서면 허가는 필요하지 않다.

다만 ‘무료 사용 가능’은 ‘조건 없이 파일을 재배포 가능’이라는 뜻이 아니다. 화면 이미지나 영상에 글꼴을 사용하기만 할 때는 일반적으로 별도 표기가 필수는 아니지만, 글꼴 파일 또는 변환한 글리프 데이터를 배포할 때는 OFL 의무를 지켜야 한다.

### 이 프로젝트의 `.FNT`는 수정본이다

BDF·TTF·OTF에서 현대 한글만 추출하고 Arena 전용 16×16 원시 비트맵 레코드로 변환하는 작업은 형식 변환과 부분 추출을 포함한다. 따라서 `HANGUL9.FNT`, `HANGUL12.FNT`, `HANGUL16.FNT`는 원본 폰트의 단순 사용물이 아니라 OFL상의 수정된 폰트 소프트웨어로 취급한다.

배포할 때 다음을 지킨다.

1. 원본 저작권 문구를 보존한다.
2. 영문 OFL 1.1 전문을 배포물에 포함한다.
3. 파생 글리프 뱅크도 OFL 1.1 조건으로 제공한다.
4. 수정 내용, 원본 버전, 원본 주소와 생성 도구를 기록한다.
5. 폰트 파일만 단독 상품으로 판매하지 않는다.
6. 폰트 저작자 이름을 프로젝트의 보증이나 홍보에 사용하지 않는다.
7. 예약 글꼴명이 있는 폰트의 파생 파일에는 그 이름을 정식 폰트명으로 사용하지 않는다.

프로젝트 코드의 라이선스와 폰트 파생 파일의 OFL은 분리한다. OFL 폰트를 번들한다고 해서 패처 코드 전체가 OFL이 되는 것은 아니다.

## 배포물의 라이선스 디렉터리

실제 사용한 폰트만 다음 구조로 넣는다.

```text
licenses/
└─ fonts/
   ├─ hangul9/
   │  ├─ COPYRIGHT.txt
   │  ├─ OFL.txt
   │  └─ SOURCE.txt
   ├─ hangul12/
   │  ├─ COPYRIGHT.txt
   │  ├─ OFL.txt
   │  └─ SOURCE.txt
   └─ hangul16/
      ├─ COPYRIGHT.txt
      ├─ OFL.txt
      └─ SOURCE.txt
```

`SOURCE.txt`에는 다음 정보를 적는다.

```text
프로젝트 글리프 뱅크 이름:
기반 폰트와 버전:
원본 제작자와 저작권:
원본 주소:
원본 파일 SHA-256:
변환 도구와 명령:
변환 내용:
출력 파일 SHA-256:
라이선스:
```

게임 안의 별도 크레딧 화면은 라이선스상 필수로 보지 않지만, README와 배포물의 제3자 고지에는 기반 폰트와 제작자를 표시한다.

## 검수 절차

폰트를 채택하거나 바꿀 때 다음을 모두 확인한다.

- 현대 한글 11,172자 존재 여부
- 원본 버전과 SHA-256
- 라이선스 전문과 저작권 문구
- 예약 글꼴명 존재 여부
- 원본 픽셀 크기와 기준선
- 320×200 화면에서 획 식별 가능 여부
- 영문 혼합 시 기준선과 간격
- UI 영역, 클릭 영역과 줄바꿈
- EMS 사용량과 화면 전환 안정성
- 배포물의 `licenses/fonts/` 포함 여부

이 문서는 프로젝트의 배포 절차를 정리한 것이며 법률 자문은 아니다.
