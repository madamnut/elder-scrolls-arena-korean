# 실행 시험 및 회귀 테스트

## 기본 실행

1. 열려 있는 DOSBox를 완전히 종료한다.
2. `Arena Korean Test (Windowed).bat`을 실행한다.
3. DOS 창에 TSR 설치 성공 문구가 나오는지 확인한다.
4. 게임이 타이틀까지 진입하는지 확인한다.

TSR은 DOSBox 프로세스 안에 상주한다. 파일만 교체하고 같은 DOSBox 세션에서 게임을 다시 실행하면 이전 TSR이 계속 사용될 수 있다.

## 빠른 연기 시험

- 원본 영어가 깨지지 않는다.
- `생성`, `선택`, `남성`, `여성`이 보인다.
- 마우스 클릭 영역과 글자가 일치한다.
- 화면 전환과 팔레트가 깨지지 않는다.
- 게임 종료 뒤 DOSBox가 정상적으로 닫힌다.

## 오프닝 순서

1. 타이틀과 저작권
2. 인용문
3. 두루마리 애니메이션
4. 줄거리 두루마리 3장
5. 메인 메뉴
6. 새 게임 줄거리 9장
7. 캐릭터 생성 방식 선택

각 화면에서 확인할 것:

- 한국어 철자
- 원본 그림 누락 여부
- 색상과 팔레트
- 화면 가장자리 잘림
- 클릭으로 다음 화면 진행

## 캐릭터 생성 질문

- 안내문이 네 줄로 정상 출력된다.
- 두루마리 위·아래 조작이 된다.
- `A`, `B`, `C` 키가 선택지와 일치한다.
- 질문과 선택지가 서로 겹치지 않는다.
- 판정 태그가 구조를 깨뜨리지 않는다.
- 무작위 10문항을 모두 완료할 수 있다.
- 결과 화면으로 전환된다.

질문은 40개 중 일부만 한 번에 나오므로 여러 번 생성 과정을 반복해 서로 다른 문항을 확인한다.

## 글꼴 회귀 시험

| 글꼴 계열 | 확인 화면 |
|---|---|
| `FONT_C` | 큰 제목 |
| `FONT_A` | 메뉴와 캐릭터 생성 선택 |
| `ARENAFNT` | 일반 게임 메시지 |
| `CHARFNT` | 캐릭터 정보 |
| `FONT_D` | 작은 설명문 |
| `FONT_B` | 직업 질문 |
| `FONT_S` | 지도·작은 UI |

각 글꼴에서 확인할 것:

- 한글 획이 식별되는가
- 행간이 겹치지 않는가
- 글자 전진 폭이 너무 넓지 않은가
- 가운데 정렬이 맞는가
- 혼합 ASCII 숫자와 문장부호가 자연스러운가

## 저장·불러오기 안전

- 패치판에서 새 저장을 만든다.
- 게임을 종료하고 다시 불러온다.
- 원본 실행기로 같은 저장을 열 때 오류가 없는지 확인한다.
- 패처 설치·제거가 세이브 파일을 변경하지 않았는지 해시나 시간으로 확인한다.

## 스크린샷

DOSBox 스크린샷 키는 `Ctrl+F5`다. 기본 저장 위치는 `DOSBox-0.74/capture`다.

보고 시 포함할 내용:

```text
패치 버전:
화면 순서:
선택한 경로:
문제 문구:
재현 횟수:
DOSBox 완전 재시작 여부:
스크린샷:
```

## 실패 분류

1. DOS 단계에서 실행 실패
2. 영어까지 깨짐
3. 한글이 빈칸·네모로 표시
4. 한글 위치·폭·행간 오류
5. 팔레트나 그림 오류
6. 입력·클릭·스크롤 오류
7. 다음 화면으로 진행 불가
8. 저장·불러오기 오류

## 릴리스 전 필수 검사

- 매니페스트 원본 해시 일치
- 설치 성공
- 중복 설치 처리
- 상태 검사 성공
- 일부 파일 삭제 후 복구 또는 오류 안내
- 원본 복구 성공
- 복구 뒤 원본 해시 일치
- 패치 ZIP을 공백·한글이 있는 경로에서 실행
- 관리자 권한이 없는 Steam 라이브러리에서 실행
- `Program Files (x86)` 설치에서 권한 상승 처리
