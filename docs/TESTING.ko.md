# 실행 시험 및 회귀 테스트

## 기본 실행

1. 열려 있는 DOSBox를 완전히 종료한다.
2. `Arena Korean Test (Windowed).bat`을 실행한다.
3. DOS 창에 TSR 설치 성공 문구가 나오는지 확인한다.
4. 게임이 타이틀까지 진입하는지 확인한다.

TSR은 DOSBox 프로세스 안에 상주한다. 파일만 교체하고 같은 DOSBox 세션에서 게임을 다시 실행하면 이전 TSR이 계속 사용될 수 있다.

## 빠른 연기 시험

- 원본 영어가 깨지지 않는다.
- `생성`, `선택`, `남성`, `여성`이 보인다.
- 마우스 클릭 영역과 글자가 일치한다.
- 화면 전환과 팔레트가 깨지지 않는다.
- 게임 종료 뒤 DOSBox가 정상적으로 닫힌다.

## 오프닝 순서

1. 타이틀과 저작권
2. 인용문
3. 두루마리 애니메이션
4. 줄거리 두루마리 3장
5. 메인 메뉴
6. 새 게임 줄거리 9장
7. 캐릭터 생성 방식 선택

각 화면에서 확인할 것:

- 한국어 철자
- 원본 그림 누락 여부
- 색상과 팔레트
- 화면 가장자리 잘림
- 클릭으로 다음 화면 진행

## 캐릭터 생성 질문

- 안내문이 네 줄로 정상 출력된다.
- 두루마리 위·아래 조작이 된다.
- 한글 획이 9픽셀 원본 형태로 식별된다.
- 한글 줄은 11픽셀 간격으로 이동하고 서로 겹치지 않는다.
- ASCII만 있는 `FONT_B` 줄은 원본 7픽셀 간격을 유지한다.
- `A`, `B`, `C` 키가 선택지와 일치한다.
- 질문과 선택지가 서로 겹치지 않는다.
- 판정 태그가 구조를 깨뜨리지 않는다.
- `a)`, `b)`, `c)`와 `(5l)/(5c)/(5v)`가 한글 셀의 아래쪽 기준선에 맞는다.
- 판정 태그가 혼자 떨어진 줄 없이 모든 질문 줄이 같은 11픽셀 간격을 유지한다.
- 무작위 10문항을 모두 완료할 수 있다.
- 한 문항의 질문과 a/b/c만 보이며 다음 번호의 문항이 같은 두루마리에 이어지지 않는다.
- 질문 두루마리를 아래로 내렸다가 올려도 다른 번호의 문항이 노출되지 않는다.
- 질문 화면에 들어오자마자 현재 문항의 첫 줄부터 표시되며, 먼저 스크롤을 움직여야만 나타나는 현상이 없다.
- 질문 줄은 전용 스크롤에서도 11픽셀 간격이고 1픽셀씩 겹치지 않는다.
- 위·아래로 한 논리 줄 이동하는 데 11단계의 부드러운 스크롤이 적용된다.
- 질문 본문이 X=310을 넘지 않고 오른쪽에 약 10픽셀 여백이 남는다.
- 결과 화면으로 전환된다.

질문은 40개 중 일부만 한 번에 나오므로 여러 번 생성 과정을 반복해 서로 다른 문항을 확인한다.
질문 빌더는 `FONT_B` 실제 폭표를 사용하고, 원본 질문이 사용하던 최대 299픽셀 폭과 126픽셀 세로 배치량을 넘으면 실패해야 한다. 현재 번역의 최장 문항 번호와 계산 높이는 빌드 로그에서 확인하고 그 문항의 아래쪽 잘림을 우선 시험한다.

## 추천 직업 결과

- 질문 10개가 끝난 뒤 본문 네 줄이 모두 한글로 나온다.
- 추천 직업명이 영어 원문으로 나온다.
- 가장 긴 `Nightblade` 또는 `Battlemage`가 오른쪽 경계를 넘지 않는다.
- 두 선택지 글자가 일반 텍스트로 각각 `예`, `아니요`로 보인다.
- 선택지 글자가 각 두루마리 칸의 가운데에 놓이고 잘리거나 겹치지 않는다.
- 수락과 거절 입력이 원본과 같은 결과로 이어진다.

## 지도 진입 전 캐릭터 생성

- 직접 선택 경로에서 `직업을 선택하라.`는 한글로, 직업명 18개는 영어로 나온다.
- 추천 수락과 직접 선택 양쪽 모두 이름 입력 화면으로 진행한다.
- 이름 입력 화면에서 선택한 직업명과 `이름은?`이 잘리지 않는다.
- 영문자와 공백으로 이름을 입력하고 확정할 수 있다.
- 성별 선택 제목과 `남성`, `여성`이 모두 한글로 나온다.

## 출신 지역 선택 지도

- 상단 리본 왼쪽에 `탐리엘`, 오른쪽에 `제국`이 나온다.
- 기본 지역명 9개가 지정 좌표에 붉은색으로 나온다.
- 출신 지역 선택 화면에서는 마우스를 올리기만 해서는 지역명이 바뀌지 않는다.
- 지역을 클릭하면 클릭 마스크와 일치하는 지역의 확인창이 열린다.
- 캐릭터 생성 이후 월드맵에서는 현재 지역의 금색 한글 지역명이 정확한 위치에 표시된다.
- 지역을 클릭했을 때 보이는 지역과 실제 선택 결과가 일치한다.
- 지도 바깥과 종료 영역의 클릭 판정이 원본과 같다.
- 성별을 확정하면 출신 지역 지도 화면으로 정상 진입한다.

## 지역 확정 후 능력치 창 진입 전

- 8개 선택 가능 지역에서 확인 질문과 `예`, `아니요`가 한글로 나온다.
- `아니요`를 선택하면 지도와 클릭 판정이 유지된 채 지역 선택으로 돌아온다.
- `예`를 선택하면 공통 안내문 네 장이 순서대로 나오며, 직업명과 입력한 이름을 제외한 영어 문장이 남지 않는다.
- 첫 번째 안내문에 직업, 이름, 선택 지역이 올바르게 삽입된다.
- 두 번째 안내문에 선택한 종족과 일치하는 `TEMPLATE.DAT` 키 1409~1416의 한글 설명이 나온다.
- 혈통 설명은 번역문에 지정한 줄 단위로 표시되며, 중간에 자동으로 다시 꺾이거나 빈 줄·겹침·오른쪽 잘림이 생기지 않는다.
- 혈통 안내 전후의 공통 안내문과 시작 컷신 자막은 기존과 동일하게 정상 출력된다.
- 세 번째 안내문에 선택한 직업과 일치하는 한글 선호 능력치 조합이 나온다.
- 가장 긴 종족 설명과 `Nightblade`, `Battlemage`를 사용해도 두루마리 밖으로 잘리거나 줄이 겹치지 않는다.
- 남성과 여성 모두 네 안내문을 통과하며, 마지막 안내문 뒤 능력치 배분 화면으로 정상 진입한다.

## 메인 메뉴

- `저장한 게임 불러오기`, `새 게임 시작`, `종료`가 새 한글 이미지로 보인다.
- 각 글자 높이 안의 왼쪽·가운데·오른쪽을 눌러도 같은 항목이 선택된다.
- Y=51~65는 불러오기, Y=104~118은 새 게임, Y=151~164는 종료로 동작한다.
- 세 범위 사이의 빈 공간을 클릭해도 다른 항목이 잘못 선택되지 않는다.
- 단축키 `L`, `S`, `E`가 원본처럼 동작한다.

## 능력치 배분과 외형 선택

- 능력치 화면 진입 안내가 한글로 나오며 화면 밖으로 잘리지 않는다.
- 기본 능력치 8개가 `힘·지·의·민·속·체·매·운` 순서로 보인다.
- 각 행을 선택했을 때 화살표가 같은 행에 놓이고, 증감한 수치가 선택한 능력치와 일치한다.
- 11픽셀 행 간격에서 라벨·수치·화살표가 서로 겹치지 않는다.
- 파생 수치, 체력·피로·골드·경험치·레벨, `완료`, `보너스 점수`가 한글로 보인다.
- `완료`를 누르면 `S 저장`, `R 다시 굴림` 선택지가 나오며 두 단축키가 원본처럼 동작한다.
- 보너스 점수를 남긴 채 저장하면 모두 배분하라는 한글 경고가 나온다.
- 모든 점수를 배분하고 저장하면 외형 선택 안내가 한글로 나온다.
- 얼굴의 왼쪽과 오른쪽을 클릭했을 때 이전·다음 외형이 정상 전환된다.
- 마지막 `완료` 뒤 리아 실메인 꿈 장면이 한글로 재생되고 시작 던전에 진입한다.
- 리아 실메인 자막은 확장한 37픽셀 밴드의 Y=166·177·188 세 행으로 표시되며, 점·끊긴 획·영상 프레임 손상이 없다.
- 한글 자막을 표시하는 동안 영상과 음악의 재생 속도가 눈에 띄게 느려지거나 끊기지 않는다.
- 자막 33행은 빈 행 없이 3행씩 정확히 11구간으로 재생된다.
- 입력한 이름이 들어가는 두 행도 화면 좌우를 벗어나지 않고 가운데 정렬된다.
- 마지막 `나를 찾아라. 너는 나의 마지막이자 최선의 희망이다...`가 표시된 뒤 시작 던전으로 정상 전환된다.

클릭 행과 화살표가 어긋나면 선택한 행, 실제 바뀐 능력치, 화살표가 나타난 행을 함께 기록한다.

## 첫 본편 리아 실메인 꿈

- 첫 본편 꿈 조건을 충족하면 `CHAOSVSN.FLC`의 혼돈의 지팡이 장면이 먼저 재생되고, 한글 15행이 세 행씩 정확히 5화면에 나온다.
- 마지막 `그곳의 이름은 팽 레어다...` 뒤 `VISION.FLC`의 리아 실메인 장면으로 이어지고, 팽 레어 안내 16행이 6화면에 나온다.
- `%pcf`와 `%pcn` 위치에 입력한 캐릭터 이름이 들어가며 그 행도 화면 안에서 가운데 정렬된다.
- 두 영상 모두 자막이 Y=166·177·188에 온전히 표시되고 점·세로선·테두리 사각형·꽉 찬 사각형·잔상이 생기지 않는다.
- `CHAOSVSN.FLC`의 지팡이 그림과 애니메이션, `VISION.FLC`의 리아 실메인 그림이 자막 위에서 깨지거나 팔레트가 바뀌지 않는다.
- 두 장면을 연속 재생하는 동안 영상과 음악이 눈에 띄게 느려지거나 멈추지 않는다.

## 후속 꿈과 사망 컷신

- 원본 영어판 대조에서 새 게임·첫 지팡이 조각 획득 전 사망은 `With you died our...`로 시작하는 리아 키 1402가 호출됨을 확인했다.
- 혼돈의 지팡이 조각 진행에 따라 `VISION.FLC`의 키 1295~1302가 한글로 나오고, 원문의 행 수와 화면 전환 시점을 유지한다.
- 진행 중 제이거 탄이 끼어드는 `JAGAR.FLC`의 키 1392~1399가 한글로 나오며 영어 자막이 남지 않는다.
- 사망 처리의 진행값 `P=[0x0F77]`이 0이면 리아 실메인의 `VISION.FLC`·키 1402, 1~8이면 제이거 탄의 `JAGAR.FLC`·키 1403이 나온다. 첫 던전 사망 자막은 `너와 함께 정의의 마지막 희망도 죽었다.`로 시작해야 한다. 리아 영상 위에 `.Perhaps I will even...` 또는 `years as Emperor`가 나오면 키 1402 본문 주소가 다시 어긋난 것이고, 영상까지 제이거 탄으로 바뀌면 실제 진행값 분기 문제다.
- 두 영상의 자막은 Y=166·177·188에 온전히 표시되고, 아래가 잘리거나 줄끼리 겹치지 않으며 점·세로선·사각형·잔상이 생기지 않는다.
- `VISION.FLC`는 20프레임·171ms, `JAGAR.FLC`는 28프레임·128ms의 원본 재생 속도와 Y=0~162 그림·팔레트를 유지한다.
- 서로 다른 키의 컷신을 연속해서 본 뒤에도 앞 장면의 음절이 점이나 사각형으로 남지 않는다. 이는 `CUTSCN.CCH`의 장면별 캐시가 공통 FLC 실행 직전에 교체됐는지 확인하는 핵심 항목이다.
- 제국 궁전 키 1447, 최종 제이거 탄 키 1404, 황제 키 1405·1407, 리아 엔딩 키 1406에 영어 자막이 남지 않는다.
- `NUJAGDTH.FLC`와 `NUKING.FLC`도 그림 Y=0~162와 원본 프레임 수·재생 속도를 유지하고, 자막은 37픽셀 밴드 안에만 나온다.

## 첫 던전과 공통 인게임 UI

- 던전 진입 직후 물방울·감방·루비 빛을 설명하는 `START.INF` 첫 문장이 한글로 나온다.
- 서쪽과 남쪽 이동, 작은 발톱 소리, 붉은 눈, 전이의 문, 안전한 휴식 틈새까지 여섯 설명이 모두 한글로 나온다.
- 루비 열쇠를 찾고 감방 문을 열 때 `a Ruby`가 남지 않고 `루비 열쇠`로 표시된다.
- 쥐·고블린 등 첫 던전 몬스터명이 한글로 표시되고, 영문 직업명은 바뀌지 않는다.
- 획득한 무기·방어구·물약 이름이 `재질 + 이름 + (마법 효과)` 순서의 한글로 표시된다.
- 잠긴 문과 상자, 자물쇠 따기 난이도·성공·실패·치명타 메시지가 한글로 나온다.
- 훔치기 버튼을 누르면 `훔칠 대상을 선택하십시오...`가 온전히 표시된다.
- 금화와 시체 수색, 빈 상자, 가득 찬 인벤토리, 장비·버리기 제한 문구가 한글로 나온다.
- 빈 고블린 시체를 확인하면 `고블린에는 쓸 만한 것이 없다.`가 나온다.
- 바닥·상자·시체에 아이템이 3개 이상 있을 때 한 페이지에는 Y=145·156의 두 항목만 나오고, 다음 페이지가 있으면 Y=167에 `...More`가 표시되어 서로 겹치지 않는다.
- 다음 페이지 이동과 수량 입력 뒤에도 선택한 항목·입력 커서가 11픽셀 행에 맞고, 마지막 항목까지 빠짐없이 접근할 수 있다.
- 휴식 시간 입력, 시간 경과, 회복, 주변의 적, 휴식 불가, 피로 탈진 문구가 한글로 나온다.
- 휴식 메뉴의 두 선택지가 단축키와 함께 `C 시간 지정 휴식`, `U 완전 회복까지`로 나오며 C/U 입력이 원본과 같이 동작한다.
- 저장 불가와 이동 불가 문구의 줄바꿈이 겹치지 않고 화면 밖으로 나가지 않는다.
- 상태 버튼을 누르면 현재 위치·시각·날짜·현재 중량·최대 중량이 한글 라벨과 실제 값으로 표시되고, 요일·달·시간대에 영문이 남지 않는다.
- 첫 던전 자동지도 상단에 `제국 지하감옥`이 나오고, `북/동/남/서`와 `종료`가 보이며 지도 선·플레이어 표시·나침반 그림이 훼손되지 않는다.
- 일지에서 `일지`, `인쇄`, `종료`가 보이고 세 클릭 영역이 원본처럼 동작한다.
- 장비 화면 상단과 하단에 `레벨:`, `장비`, `종료`, `마법서`, `버리기`가 보이고 클릭 기능이 원본과 일치한다.

## 마을 주민 대화

- 마을에서 주민에게 말을 걸면 `질문 선택`과 `W 누구인가?`, `H 장소 묻기`, `R 소문`, `E 끝`이 한글로 나온다.
- W/H/R/E 단축키와 마우스 선택이 원본과 같은 분기로 이어진다.
- `장소 묻기`를 선택하면 여관·사원·장비점·마법사 길드·궁전·성문 또는 일거리와 가까운 시설 목록이 모두 한글로 나온다.
- 장소를 골랐을 때 위치를 모르는 답, 8방향 안내, 지도에 직접 표시하는 답이 모두 한글로 나오고 북·북동·동·남동·남·남서·서·북서 방향이 실제 위치와 일치한다.
- 주민의 도시별 인사말이 한글로 나오며 현재 도시와 맞는 지명·내용을 표시한다.
- 도시 64곳의 인사말은 원문과 같은 행 수로 나오고, 줄 겹침·오른쪽 잘림·다음 레코드 혼입이 없다.
- 소문을 선택하면 `소문 종류`, `G 일반`, `W 일`이 나오고 G/W가 각각 올바르게 동작한다.
- 같은 주민에게 `누구인가?`를 두 번 물어 첫 만남 키 `0100`~`0114`와 재대화 키 `0115`~`0129`가 모두 한글로 표시되는지 확인한다.
- 역할별 대사를 반복해 같은 역할 안에서도 친절·냉소·모욕 변형이 원문 수위에 맞게 달라지고, 성매매 여성·마법사·거지·음악가·노예상의 말투가 서로 구분되는지 확인한다.
- 공용 소문·일거리·장소 안내는 어떤 외형의 주민이 말해도 화자의 성별이나 직업과 충돌하는 표현이 없는지 확인한다.
- 행상인·상인·마법사·사제·음악가·거리 청소부 등 서로 다른 외형의 주민에게 물어도 이름·직업·도시 자리표시자가 깨지지 않는다.
- `일반`과 `일`을 여러 주민에게 반복해 무작위 답변을 확인한다. 소문 없음 키 `0159`, 일거리 없음 키 `0160`, 도시 사건 키 `0162`~`0168`, 축일 조각 키 `0169`~`0180`에서 영어가 섞이지 않는다.
- 대화할 내용이 없거나 위치를 모를 때의 답변과 소문 거절문도 영어 없이 표시된다. 답변이 다음 레코드로 넘어가거나 화면 오른쪽에서 잘리지 않는다.
- 장비 구입·매각·여관 숙박·마법사 길드 거래에서 첫 제안, 수락, 너무 낮거나 높은 제안, 거래 결렬, 절충안의 다섯 단계가 모두 한글로 나온다.
- 같은 상인 화법은 거래 단계가 바뀌어도 서툰 말투·거친 구어체·정중한 말투·오만한 말투가 일관되고, 원문 욕설과 모욕 수위가 임의로 약해지거나 강해지지 않는다.
- 주점의 일반 손님을 반복해서 눌렀을 때 51개 거절·농담·주인장 안내 문안이 겹치거나 영문과 섞이지 않는다.
- 장비 상세의 `상태`, `피해`, `방어도`, `무게`, 남은 충전·사용 횟수가 겹치거나 잘리지 않는다.
- 일반 메시지에 사용되는 `%s`, `%ss`, `%u`, `%d` 값이 숫자·이름과 함께 정상 표시된다.

환경 설명과 기본 공통 메시지는 런타임 통과했다. 새 이름 표와 장비 화면은 런타임 시험 대기다. 실패하면 문구 원문, 진입 경로, 사용한 아이템·열쇠명, 화면의 줄 수를 함께 기록한다.

## 글꼴 회귀 시험

| 글꼴 계열 | 확인 화면 |
|---|---|
| `FONT_C` | 큰 제목 |
| `FONT_A` | 메뉴와 캐릭터 생성 선택 |
| `ARENAFNT` | 일반 게임 메시지 |
| `CHARFNT` | 캐릭터 정보 |
| `FONT_D` | 작은 설명문 |
| `FONT_B` | 직업 질문 |
| `FONT_S` | 지도·작은 UI |

각 글꼴에서 확인할 것:

- 한글 획이 식별되는가
- 행간이 겹치지 않는가
- 글자 전진 폭이 너무 넓지 않은가
- 가운데 정렬이 맞는가
- 혼합 ASCII 숫자와 문장부호가 자연스러운가

## 저장·불러오기 안전

- 패치판에서 새 저장을 만든다.
- 게임을 종료하고 다시 불러온다.
- 원본 실행기로 같은 저장을 열 때 오류가 없는지 확인한다.
- 패처 설치·제거가 세이브 파일을 변경하지 않았는지 해시나 시간으로 확인한다.

## 스크린샷

DOSBox 스크린샷 키는 `Ctrl+F5`다. 기본 저장 위치는 `DOSBox-0.74/capture`다.

보고 시 포함할 내용:

```text
패치 버전:
화면 순서:
선택한 경로:
문제 문구:
재현 횟수:
DOSBox 완전 재시작 여부:
스크린샷:
```
